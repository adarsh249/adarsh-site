CSE 135 - Online Database Analytics Applications

This class was about collecting data and outputting in a format that is readable. Emphasis was placed on what kind of data we should be collecting, how much data we should collect and what are the best ways to present this data. This class went very in depth into data visualization. We had four assignments that constantly built upon each other, but the end goal is what is seen on the resume. Using Apache and Digital Ocean we, a group of 3, first created our own site and domain. Then, as a mini assignment, we did learn about CGI and other programming languages. In total, we used Perl, C, JSP, Python, Ruby, PHP and a couple other languages to learn about sessioning and other things. However, the bulk of what we learned were in assignments 3 and 4. We created a collector.js script and in this script we collected user's static, performance and activity data. In other words, any time a user entered our site, we gave them a session ID and checked everything about their browser, computer, connection, device type, etc. We also checked their load time, how long it took to load the page, what time they entered the page, etc. For activity data, we collected the cursor positions on the page at all times, keys pressed, when the user left our page, when they came back, how long they were idle for, if they went idle at all. We checked numerous things. Then, using Node.js, we would send POST requests to our custom routes we created which would then store this data in a MySQL database. You could also go to our site and see the data in JSON readable format. Finally, using ALL this data, we made multiple graphs, (line charts, pie charts, bar charts) and made a complete analytical dashboard. In our case, we focused on the relationship between device type, page loading time, and network connection. We created a full on dashboard where users could see this information in a readable format. In order to access this dashboard, you must have been a registered user. We created a login form, using JWT tokens. Non-users could access nothing. Normal users had access to the dashboard and a two-page report about the dashboard. Admin level users had access to both these things in addition to a user management table where admins could create, update and delete users. The majority of this whole operation was done in Node.js, with the graphs being done in PHP extracting data from a MySQL database. The rest of the site was built on HTML and CSS. Charts were created using ZingChart and the management table was created using ZingGrid.
CSE 110 - Software Engineering

In this class, you are randomly placed in groups of 10 and assigned a role. I happened to be assigned team lead. This class focuses on the process of software engineering and how you can get from start to finish of an application. The theme this quarter was fortune telling and all resources, including AI and ChatGPT, were allowed to be used. I'll break this class down through a week by week process. In week 1 and 2, there was the introduction to the class and getting to know your team members better. This class follows the Agile process directly. As a team lead, I set up daily stand ups, led the weekly team meeting and provided random announcements through the Slack channel. I also contributed to the code base. In week 3, we began brainstorming and created user stories and design. Then, in week 4, we began creating our spec and thought of process models, diagramming basic flow charts, and having minimum requirements. In week 5, we created wire frames and designs of what we wanted the final product to look like. Then, from weeks 6 to 10, the coding process began along with testing nearing the end. Coding practices included having a robust CI/CD pipeline, having weekly sprints and weekly retrospectives after the sprints as well. Git was utilized extensively and every team member by the end got used to working together in an environment around git and GitHub. You can check out the end product here. There is lots to see on the GitHub which can be accessed upon request.

This was one of my favorite classes. In this class, we went over data structures in C++. We learned the best, worst, and average case for all data structures and implemented our own as well. Data structures included trees, graphs, priority queues, hash tables. We also learned about memory management, pointers, and recursion and created a couple of fun projects at the end.

Projects included a basic alarm clock with AM/PM and the ability to set an alarm on any day of the week, month, and year. We also recreated the infamous Simon Says game as well as ended the class by creating an encryption service where input text would be encrypted.
